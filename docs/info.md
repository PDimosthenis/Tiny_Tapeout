# 16-bit MIPS Single Cycle Processor

This project implements a small 16-bit MIPS-like single-cycle CPU for Tiny Tapeout.

## Overview

The CPU executes 16-bit instructions, but instructions are loaded through the existing 8-bit `ui_in` pins by writing the low byte and high byte separately to the same instruction address.

## Main Features

- 16-bit datapath and 16-bit registers
- 16-bit instruction words
- 4 x 16-bit register file
- 4-word 16-bit data memory
- Single-cycle execution
- Load/run mode controlled through `uio_in[7]`

## Pin Usage

- `ui_in[7:0]`: instruction byte during load mode
- `uio_in[7]`: mode select, `0` = load, `1` = run
- `uio_in[6]`: byte select, `0` = low byte, `1` = high byte
- `uio_in[3:0]`: instruction address during load mode
- `uo_out[7:0]`: lower 8 bits of ALU result in run mode
- `uio_out[3:0]`: current PC in run mode

## Load Flow

To load one 16-bit instruction:

1. Set `uio_in[7] = 0` for load mode.
2. Put the target instruction address on `uio_in[3:0]`.
3. Write the low byte on `ui_in` with `uio_in[6] = 0`.
4. Write the high byte on `ui_in` with `uio_in[6] = 1`.
5. Switch `uio_in[7] = 1` to run the CPU.

## Notes

- The register file only uses 4 architectural registers, selected by the low 2 bits of the decoded register fields.
- `MemRead`, `MemWrite`, `RegWrite`, `ALUsrc`, `RegDst`, `MemtoReg`, and `Jump` are control signals generated by the control unit to steer the datapath.
